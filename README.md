# **GotoBot**
## О GoToBot
Жизнь в GoTo насыщена большим количеством интересных событий. Семинары, лекции, квесты, игры - все это часто в разных группах, а ведь иногда нужно и поесть не забывать. 
GotoBot является ботом помощником для обучающихся и учителей в GoToCamp. Он способен автоматически сообщать о событиях происходящих в лагере, с его помощь можно легко узнать расписание на день, контактную информацию жителей, список "ачивок" человека и пройти квест.
При этом учителя и организаторы могут удобно администрировать все события, перейдя в специальный режим c помощью ввода пароля.
Одной из главных целей во время написания проекта (который, разумеется, будет дорабатываться), была возможность пользователя легко и интуитивно понятно взаимодействовать с ним.

## /admin
Основная и, наверное, вторая по значимости команда, которая позволяет пользователю получать права администратора, добавлять ачивки, мероприятия, посылать срочные сообщения и ДАЖЕ сменить язык для всех пользователей бота!)
Переход в режим администратора происходит, используя пароль, по умолчанию пароль задан master 000000
>/admin master 000000

Переход в режим администратора:

![Bot admin](https://github.com/AlexNerru/GotoBot/blob/develop/Screenshots/photo_2017-03-19_13-55-58.jpg?raw=true)

## Стандартные команды
Команды доступные обычным пользователям, основной функционал.

### /add_phone и /add_address
Служат для добавления и изменения информации о вашем телефоне и адресе. Это нужно, чтобы в случае чего, ученики, вожатые или преподаватели могли вас найти.
>/add_phone /add_adress

Служат для поиска информации о человека, по имени и фамилии находится телефон или адрес, который пользователь ввел командами '/add_phone' и '/add_address'

## Про ошибки и баги
Большинство ошибок на неверный ввод, последовательность команд и прочее я стараюсь отлавливать, может случится, что появится ошибка, которая необработана, прошу писать мне на Github или в Telegram в таком случае. Могут быть ошибки из-за большого числа пользователей, которые взаимодействуют с определенным модулем одновременне, но я старался их максимально избежать.

Обрабатываются ошибки:
1. Неверный ввод
2. Ошибки бд
3. Одновременный ввод
4. Отправка команды вместо текста
5. Другие

## Про установку
Скачиваем репозиторий и запускаем EchoBot.py (да, название с начала разработки я так и не поменял) через Pycharm например, в остальном проблем возникнуть не должно. Необходим Pyhton 3.6, поддержка sqlite и выход в интернет.
